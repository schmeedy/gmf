The commands that implement the MSL undo/redo command stack. These are used internally by the MSL to implement undo/redo support in EMF models, and by clients to inject undo/redo logic associated with model changes into the MSL's command stack to coordinate related changes in some other non-EMF data store.

Package Specification

The core types in this API are:

  • {@link org.eclipse.gmf.runtime.emf.core.internal.commands.MCommand}: abstract specification of an MSL command, extending the EMF {@link org.eclipse.emf.common.command.Command} interface
  • {@link org.eclipse.gmf.runtime.emf.core.internal.commands.MSLUndoStack}: the undo/redo stack of MCommands
  • {@link org.eclipse.gmf.runtime.emf.core.internal.commands.MSLAbstractCommand}: abstract partial implementation of the MCommand interface
  • {@link org.eclipse.gmf.runtime.emf.core.internal.commands.MSLCompoundCommand}: implements composition of MCommands into single undoable units

The {@link org.eclipse.gmf.runtime.emf.core.internal.commands.MSLCommandGenerator} is not a part of this package's API; it is used internally by the MSL to populate the command stack in response to notifications from EMF.
