HowTo … ?

 

  1. How to obtain a reference to CommandManager ?

 

Every ActionManager has a CommandManager.  You can obtain a reference to ActionManager from am Aurora part (view or editor).  Once you have a reference to ActionManager, you can obtain a reference to CommandManager.

 

[back to top]

 

  1. How to execute a Command ?

 

All Commands should be executed using CommandManager.  However, if a CommandManager is busy doing something, the client should call execute() on  a Command directly.

 

[back to top]

 

  1. How to undo a Command ?

 

All Commands should be executed using CommandManager.  However, if a CommandManager is busy doing something, the client should call undo() on  a Command directly.

 

[back to top]

 

  1. How to redo a Command ?

 

All Commands should be executed using CommandManager.  However, if a CommandManager is busy doing something, the client should call redo() on  a Command directly.

 

[back to top]

 

  1. How to execute an AbstractModelCommand ?

 

All Model Commands should be executed through CommandManager.  Model Commands are executed in a single undo interval.  Opening of read/write actions and management of undo intervals is done silently.

  1. How to get all the work done to support a single user gesture into one “undo” Action in the Edit menu ?

 

Compose your Commands into a CompositeCommand.  For  example, if a user gesture corresponds to multiple commands, those commands should be executed as a Composite Command.

  1. How to monitor progress of a command ?

 

You can monitor the progress of a command by executing a command with ProgressMonitor.

Please read API documentation for IProgressMonitor and SubProgressMonitor.

When … ?

 

1.   When should I use Command Infrastructure ?

 

You can use the command infrastructure whenever you would like to organize user gestures so you can present them to the user for undo/redo.

 

[back to top]

 

2.   When should I use an AbstractCommand versus an AbstractModelCommand ?

 

Use AbstractCommand for commands that do not perform model operations.

Use AbstractModelCommand for commands that perform model operations such as creating, deleting and moving elements.  Model commands are executed in one undo and one redo action/

 

[back to top]

 

3.   When should I use CompositeCommand vs CompositeModelCommand ?

 

Use CompositeCommand for composing commands that do not manipulate a model.

Use CompositeModelCommand for model commands.

 

[back to top]

 

What … ?

 

  1. What can I put in the “affectedObjects” list in my command ?

 

‘affectedObjects’ in Aurora is interpreted as ‘affectedFiles’.  The client should ensure that all files in ‘affectedObjects’ are writeable.

 

[back to top]

 

  1. What are the “unwritten rules” (pitfalls) about writing and executing commands ?

 

Always check if the CommandManager is not currently executing another command before asking it to execute another command.

 

[back to top]

 

Can I … ?

 

  1. Can I put an AbstractCommand inside a CompositeModelCommand ?

 

You can put an AbstractCommand inside a CompositeModelCommand only if your AbstractCommand does not implement any undo() behavior because the CommandManager will only undo its undo interval (meaning that the undo() method in your AbstractCommand will never be called).
