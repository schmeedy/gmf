EMF & OCL Domains: Query Services (Cont.)
§ Condition and ConditionPolicy
4 Provides support for the basic EObject based Condition
objects that are used to formulate queries for EMF models.
4 The EObjectCondition class is the abstract parent of
all conditions that deal with model-elements (i.e.,
EObjects). It incorporates the services of a
PruneHandler in order to answers whether or not to
prune the element tree at a specific element and thus
ignore its children.
4 The ConditionPolicy class is used to allow the user to
decide how to apply a given condition on a collection of a
model-element EAttributes or EReferences values.
Supports both the: exists (ANY) and for-all (ALL)
semantics.
4 The EObjectStructuralFeatureValueCondition
class is the parent class for conditions that are responsible
for checking the values held in model-elements’ attributes
or references. It uses the help of two other condition
objects. The first is an EObjectCondition that it used to
decide on the validity of context element (a context
element is the model element whose attribute we are
evaluating) and the other condition is an evaluator
Condition that it uses to evaluate the value held in the
attribute or the reference and whether or not it matches
what the user is looking for
