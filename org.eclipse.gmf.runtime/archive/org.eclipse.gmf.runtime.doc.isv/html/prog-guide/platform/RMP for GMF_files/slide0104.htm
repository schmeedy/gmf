GMF Requirements Support: Diagram Definition
§
§
•A diagram definition may be mapped to zero or more domain models. In other words, a diagram may contain elements which represent information from multiple domain models. [H] The diagramming meta-model included in this submission supports any number of domain models
•A diagram may contain multiple references to a single domain model element, potentially with each having a different diagram representation. [M] Supported and productized
•A diagram may have elements added from other referenced diagram definitions. [M] Supported and productized
•Metamodels
•A diagramming metamodel will be provided to allow for diagram definitions. [H] [?] RMP’s “notation metamodel” is an EMF-based meta-model describing diagrams
•A mapping metamodel will be provided to allow for diagram to domain model mapping definitions. [H]
•Diagram groups
•Diagram definitions should support a logical grouping of diagrams, as required by the UML, for example. [M]  Support notion of diagram type
