Plug-in conventions/API conventions
§ Follows Eclipse naming conventions
4 Plugins are separated into .core and .ui for engine vs ui components
4 .ide for non-RCP supported plugins
4 Interfaces start with “I”
4 Anything that is in a package with “.internal” should be considered internal
4 All APIs have javadoc
4 Some APIs are “provisional”
4 JUnits are in .tests plugins
§ Domain dependencies are respected
§ .core plug-ins don’t depend on .ui plugins
4 .ui plug-ins are optional
