EMF & OCL Domains: Model Management Services (cont.)
4 Batched Notification Support
• Centralized model for event notifications
• Includes event filters
• Subclass MListener to handle EMF events and
implement onEvent. A listener can have a filter. Listening
starts once startListening is called
• Subclass MFilter and implement matches to control the
events sent to listeners. MFilter defines some pre-
defined filters to be composed to build complex filters.
• The listeners are associated with a editing domain
• The IDemuxedMListener interface can be implemented
by clients interested in individual resource and object
lifecycle events.
• The DemuxingMListener class is a demultiplexing event
listener, which relays batched events from the MListener
protocol one at a time, dispatching them to the appropriate
methods of an IDemuxedMListener according to the
nature of each event.
• Some of the custom event types supported are
EventTypes.CREATE, EventTypes.DESTROY,
EventTypes.IMPORT, EventTypes.UNRESOLVE and
EventTypes.EXPORT
