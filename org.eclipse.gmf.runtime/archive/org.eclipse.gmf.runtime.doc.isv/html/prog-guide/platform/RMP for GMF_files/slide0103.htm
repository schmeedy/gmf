•Extensibility
•GMF must adhere to the extensibility best practices of Eclipse.  The generated plug-ins, the runtime framework, the genmodel, generator, wizards, etc. should all be designed/implemented with extensibility in mind. [H] Service infrastructure built on top of Eclipse extension points used throughout the platform
•Interoperability
•GMF has the potential to interoperate with a number of other Eclipse projects and, potentially, support external standards and tooling frameworks. See the Interoperability section below.
Product offerings based on the platform have successfully been integrated UML2, JDT, PDE, CDT, Team among others
•Compatibility
•GMF should leverage the latest (pure) version of platform and dependent projects, Java edition, utilize MANIFEST.MF files, etc. [H] [?] Built on Eclipse 3.1
•Testing
•Unit tests will be written and maintained for appropriate code segments. [H] Major subsystems have unit tests
•GMF should support and be tested on all the platforms supported by Eclipse or, minimally, by the EMF and GEF projects. [H] Productized on Windows 2K,XP and Linux RH ELS, SUSE Linux ES, internally used on OS X
•Documentation
•Standard help, exsd files, tutorials, and samples will be provided. [H] The submission includes an SDK
•Internationalization/Localization
•GMF should be localized to the main locales supported by the platform or, minimally, by EMF and GEF. [H] Support and productized with all languages supported by EMF
•Performance
•The GMF runtime, designer, and generated components should be tested for performance characteristics and follow published guidelines. [H] Diagram runtime performance succeeded in production environment
