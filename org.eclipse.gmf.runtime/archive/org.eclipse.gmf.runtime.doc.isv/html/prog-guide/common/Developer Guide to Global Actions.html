This document describes how to add support for global actions and complete the following two use cases:

  1. Add support for new Eclipse global action such as cut, copy and paste.
  2. Add support for new Aurora global action.

  1. Use Cases

 

6.1  Add support for new Eclipse global action handler

 

1.      Add c.i.x.common.core.ui to a list of dependencies of your plugin.

2.      Declare an extension to com.rational.xtools.common.ui.globalActionHandlerProviders

3.      Extend AbstractGlobalActionHandlerProvider class:

4.      Extend AbstractGlobalActionHandler class

a.       Implement the getCommand()

b.      Implement canHandle() method

 

[back to top]

 

6.2  Add support for new Aurora global action handler

 

1.      Extend GlobalAction class.

a.       Implement the getActionId method.

b.      Depending on the requirements other methods can be overridden.

2.      Update the class GlobalActionId to include an id for the new global action.

3.      Update the schema file globalActionHandlerProviders.exsd to include the new action id as part of the constraint.
